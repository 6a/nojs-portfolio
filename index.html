自己紹介

ゲームが最高で世界一のエンターテインメントだと心より信じております。幼いころからゲームを人生の中心として楽しんできた私は将来、優れたチームと共に何千もの人々のために喜びや感動、カタルシスまでももたらすことができるゲームのものづくりに参加したいと思います。

皆さんこんにちは、スタントン・ジェームズ瑛之助です！イングランド育ちの２０２０年 Kingston（英国）の大学卒業生（科学、技術とコンピューティング学科 コンピューターゲームプログラミング学士）で、入学前に４年間 IT 企業で勤めていた、プログラミング専門のゲーム開発者です。当ポートフォリオサイトをご覧いただきありがとうございます！

物心ついて以来、私はものづくりが大好きでした。初めはデュプロ、それからレゴやケネックスへ進行しました。成長するがてら、興味がテクノロジーへと進化し、パソコンを作ったり調整したり、いじったり、壊れたエレクトロニクスを分解して再構成したりする趣味を培ってきたきました。高校卒業後、しばらくの間 IT 企業で勤めて、色々の役目を果たしながら豊かな経験と知識を手に入れたつもりですが、いつか何かを失った、もしくは何かが欠けていると思うようになりました。いつの間にか私の人生から「物を造る」チャンスが完全になくなりました。この思いをきっかけに、プログラミングを独学し始めてから一年後、最寄りの大学に入学しました。大学三年の間（２０１８年０６月～２０１９年０６月）インターンとして SONY で勤め、只今大学の課題を全て提出した、成績の発表を待ちかねている、日本のゲーム企業の就職希望者です。

詳しいプログラミング言語を手短にまとめると、C++、C#、Go, JavaScript, Python となります。ゲームエンジンなら Unreal Engine 4 と Unity、及び C++ で動いている大学の教育用特注エンジンで開発する経験があります。エンジンやゲームプレイプログラミングは大学生時代のニーズによって強みとなりましたが、いつも好奇心をもってどんなチャレンジが待ち受けているとしても、前向きに受けて立ちます。物語の創造、ゲームデザイン、3D モデリング、何でもあろうと、ゲームクリエーションのあらゆる要素が非常にユニークで魅力的で、やりがいに満ちていると思います！

私自身、ジェームズという存在を簡単にイメージすると、「完璧主義者の頑張り屋」です。自ら決めた目標も、仕事の業務だとしても、「必ず成功したい！」とひたすら思うままです。IT 企業で心得た知識、経験、態勢や平常心と、大学で養った技術やゲーム開発のノーハウを、SONY で身につけた実践体験を合わせて、更なるチャレンジをゲーム好きの同志たちと一緒に挑戦したいと思います。向上しながら、ゲームへの愛と情熱を活かして、楽しく、勢いよく、素晴らしい作品を作り上げたいと思います。

御社において、自分の強みを活用し、未経験と未熟さを「精進あるのみ！」の意志で取り払い、御社に役立つことを期待していただきたいと思います。多くのファンがそれぞれの形に喜べるゲームを制作できるようになることを必ず証明します。

私の技能や実力を示すために、前に開発したプロジェクトの中から多少の注目すべきプロジェクトをかいつまんでまとめました。更に詳しく知りたい方はぜひ、フッターに含まれているいずれかの連絡先へ遠慮なくご連絡ください。

なお、お客様のネットワーク通信に余計な負担を掛けないよう、全ての動画に強い非可逆圧縮を適用させて頂きました。劣化された画質によって細かいところが不明瞭になったり、色が薄くなった状態ですが、ご了承ください。

Blade II Online（ブレード・ツー・オンライン）は大学４年生の時に開発した、Unreal Engine 4・C++、AWS、Go、Electron・JavaScript・HTML・CSS を活用した、一対一のオンラインカードゲームです。英雄伝説 閃の軌跡 II、東京ザナドゥ、という日本ファルコムの作品に存在するミニゲームに基づいています。Blade II Online は最終のプロジェクトであったので、開発終了後、学位論文も作成しました 。

このプロジェクトの目的は、ブレードというゲームをオンライン化できるかどうかを調査し、実現したらどのようなプレイ感になるか、本当に楽しいかどうか、みたいな質問に応じることでした。結局、実装するには成功しましたが、楽しいかどうかはまた別問題です。

上記の動画はアカウント登録、ログイン、マッチメイキングの過程を示し、その後２人のプレイヤーがネット上でマッチをすることを示しています。

ご覧の通り、キャラクターの 3D モデル、エフェクト、キャラクター・テーブルのテキスチャーとその他は、Steam 版の「英雄伝説・閃の軌跡II」から直接抽出させて頂きました。

Blade II Online は４つの主要コンポーネントによって構成されています：

  • Amazon EC2 インスタンスにホストされ、Amazon RDS インスタンスに支えられ、NGINX にネットに公開され、Cloudflare にリバースプロキシされた Go で作成したマッチメイキング兼ゲームサーバー。
  • Go で作成した AWS Lambda をネットに公開する Amazon API Gateway を活用した REST API。
  • JavaScript、HTML、CSS で作成した Electron フレームワークを活用したラーンチャーアプリケーション。
  • Unreal Engine 4 と C++ で作成したゲームクライエント。

フィーチャー概要

  • マッチ歴、プロフィール、ハイスコアランキングを含まれる Elo ランキングシステム。
  • REST API 対応のアカウント登録・ログインシステム。
  • トーケンベース認証システム。
  • ラーンチャーとゲーム内の設定システム。
  • コンテキストに基づく吹き出しと口パクできるアニメーションしている敵のアバター。
  • C++で作成した特製アニメーションシステムで動かされているカードのアニメーション。
  • WebSocket 通信でのオンラインプレイ。
  • WebSocket 通信でのマッチメイキング。
  • ３つのマッチの類：チュートリアルマッチ、対 AI マッチ、オンラインマッチ。
  • UMG UI Designer で作成した 2D アニメーション。
